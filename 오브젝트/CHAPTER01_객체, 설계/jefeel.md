# 챕터 1

- Paradigm = “관점”
- 객체지향 Paradigm은 객체지향 프로그래밍을 위한 “표준”
- 티켓 판매 어플)
    - 이해하기 어려운 코드
    - 변경이 어려운 코드 → **의존성 문제**
    - ∴ 의존성을 없애 결합도를 낮춘다
    - ∴ 자율성을 높여주자 (각각의 메서드에 역할을 분리, 구현)
- 캡슐화를 통해 객체 내부에 묶어 응집도를 높인다
- 절차지향 vs 객체지향  : **책임 문제**
- ※ 의인화 관점의 객체?
- 설계의 중요성 : ********"오늘 기능 구현" + "내일 변경한 코드"********
